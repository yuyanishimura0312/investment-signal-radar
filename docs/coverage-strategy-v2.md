# VCデータベース カバー率改善戦略 v2（2016-2026全期間、80%目標）

## 調査日: 2026-04-23
## ステータス: Phase 1実証済み、全期間戦略策定

---

## 1. 実証結果（2023年1月バックフィル）

2023年1月のGoogle Newsバックフィルを実行し、以下の実績を得た。

- **入力**: 52クエリ × `after:2023-01-01 before:2023-01-31` → 342記事取得
- **出力**: 61件の新規資金調達ラウンド（重複除外済み）
- **カバー率**: 8% → **34%**（単月、INITIAL基準）
- **所要時間**: 約9分、APIコスト約$0.10
- **精度**: Claude Haiku抽出により企業名・金額・ラウンド種別を構造化

この結果から、Google News日付フィルタ検索は過去データの補完に極めて有効であることが確認された。

## 3. 戦略構成

80%カバー率の達成には、単一ソースでは不可能であり、5つの収集層を段階的に積み上げる「5層戦略」を採用する。

```
Layer 5: 有償データ連携（STARTUP DB API）        → +20-30%ポイント
Layer 4: 公的データ（官報/登記/助成金DB）          → +5-10%ポイント
Layer 3: VC Portfolio逆引きスクレイピング         → +10-15%ポイント
Layer 2: 専門メディアクロール（STARTUP LOG等）     → +5-8%ポイント
Layer 1: ニュース検索バックフィル（Google/DDG）     → +15-25%ポイント
──────────────────────────────────────────────────
Layer 0: 既存データ（PR TIMES RSS/スクレイピング）  → 6-13%（現状）
```

### 3.1 各層の詳細

#### Layer 1: ニュース検索バックフィル（実装済み、即日実行可能）

**Google News日付フィルタ検索**
- 実装: `collect_google_news.py --backfill` （52クエリ × 月別`after:/before:`）
- 対象: 2016-01 〜 2026-04（124ヶ月）
- 推定取得: 年200-700件の新規ラウンド（年代により変動）
- 所要時間: 約8時間（124ヶ月 × 52クエリ × 5秒レート制限）
- コスト: Claude API ~$12

**DuckDuckGo月別検索**
- 実装: `collect_ddg_funding.py --backfill` （52クエリ × 月名埋め込み）
- Google Newsと異なるインデックスを持つため、追加のユニーク記事を発見可能
- 推定追加取得: Google Newsの30-50%程度の上乗せ
- 所要時間: 約14時間（8秒レート制限）
- コスト: Claude API ~$8

**Layer 1合計の期待効果:**

|---|---|---|---|---|---|

#### Layer 2: 専門メディアクロール（1-2週間で実装可能）

**STARTUP LOG（startuplog.com）**
- 資金調達ニュース専門メディア。記事アーカイブあり、日付順にページネーション可能
- Playwright MCPでアーカイブページを巡回、記事URLを収集→Claude抽出
- 推定: 年200-300件のユニーク記事（2020年以降）
- 2016-2019年はカバーが薄い可能性

**The Bridge（thebridge.jp）フルクロール**
- `/tag/fundraising/`アーカイブの全ページクロール
- 英語/バイリンガル記事が多く、海外VC参画案件に強い
- 推定: 年100-200件（2018年以降）

**note.com スタートアップ発信**
- 資金調達を自社noteで発表する企業が増加（特に2022年以降）
- Google News検索に`site:note.com`を追加するだけで部分対応可能
- 推定: 年50-100件（2022年以降）

**Layer 2合計の期待効果:** +5-8%ポイント（2020年以降の年に効果集中）

#### Layer 3: VC Portfolioスクレイピング（2-3週間で実装可能）

INITIALも採用している「逆引き」アプローチ。VCのWebサイトからポートフォリオ一覧を取得し、投資先企業を特定する。

**ターゲットVC（上位50社）:**
- メガVC: JAFCO, SBI Investment, Global Brain, Genesia Ventures, Incubate Fund
- 独立系: ANRI, Coral Capital, East Ventures, DNX Ventures, THE FUND
- CVC: Sony Innovation Fund, NTTドコモ, 三菱UFJ, KDDI, 住友商事
- 政府系: DBJ Capital, INCJ, NEDO関連
- 海外: Sequoia, 500 Global, Y Combinator（日本投資分）

**手法:**
1. 各VCのポートフォリオページをPlaywright MCPでクロール
2. 企業名・投資時期・ラウンド情報を構造化
3. 既存DBとのマッチング・重複除去
4. 未登録企業をGoogle/PR TIMESで追加検索し詳細取得

**推定効果:** 上位50社で年間投資の60-70%をカバー。年200-400件のユニーク追加。
ただし、投資時期（年月）がポートフォリオページに記載されていない場合が多く、その場合は別途ニュース検索で補完が必要。

**Layer 3合計の期待効果:** +10-15%ポイント（全年代に効果あり、特に2016-2020に有効）

#### Layer 4: 公的データ（中期、1-2ヶ月）

**官報・登記情報**
- 第三者割当増資は商業登記に反映される
- 官報決算データベース（catr.jp）が一部データを構造化
- 法人番号API（houjin-bangou.nta.go.jp）は無料だが、増資の詳細は取得不可
- 課題: 非構造化テキストからの抽出が必要、NLP/Claude処理のコストが大きい

**NEDO/JST/経産省助成金データ**
- SBIR推進プログラムの受託企業リスト
- スタートアップ向け助成金の採択企業情報
- 直接的な資金調達ではないが、エコシステムの網羅性向上に寄与

**Layer 4合計の期待効果:** +5-10%ポイント（特にPR非公開の小規模ラウンドに有効）

#### Layer 5: 有償データ連携（交渉次第）

**STARTUP DB API（最有力）**
- 24,000社以上のスタートアップデータ、REST API提供
- 2025年11月にCrunchbaseと提携、国際データとも連携
- 問い合わせ先: lp.startup-db.com/api
- 推定費用: 月額5-15万円（研究用途、要交渉）
- 効果: 2020年以降のカバー率を一気に70-80%に引き上げ可能

**Crunchbase API（補完）**
- STARTUP DBと提携後、日本データが拡充
- グローバルラウンド（海外VC参画）に特に強い
- 推定費用: 月額$99-499

**Layer 5の効果:** +20-30%ポイント（Layer 1-4との組み合わせで80%達成）

## 4. 年代別の現実的な到達見込み

Layer 1-4（無償ソースのみ）で達成可能なカバー率と、Layer 5追加後の見込み：

| 年代 | Layer 1後 | +Layer 2-3 | +Layer 4 | +Layer 5 |
|---|---|---|---|---|
| 2016-2017 | 9-10% | 15-20% | 20-25% | **40-50%** |
| 2018-2019 | 12-14% | 25-30% | 30-35% | **55-65%** |
| 2020-2021 | 14-15% | 30-35% | 35-40% | **65-75%** |
| 2022-2023 | 31-38% | 45-50% | 50-55% | **75-85%** |
| 2024-2026 | 20-38% | 40-50% | 50-55% | **75-85%** |

**結論:** 80%は2022年以降に限定すればLayer 5込みで達成可能。2016-2019年は構造的に40-65%が上限（当時のオンライン公開率が低い）。全期間平均では**Layer 1-4で35-40%、Layer 5込みで65-75%**が現実的な到達点。

## 7. 主要リスクと対策

| リスク | 影響 | 対策 |
|---|---|---|
| Google Newsのレート制限 | バックフィル中断 | 5秒間隔を維持、429エラー時は指数バックオフ |
| 古い記事のリンク切れ | 2016-2019の取得率低下 | Google News URLから直接抽出（本文不要でもtitle+summaryで対応） |
| 重複ラウンドの発生 | データ品質低下 | url_hashによる重複排除 + 企業名×日付の fuzzy matching |
| STARTUP DB APIが高額/不可 | Layer 5断念 | Layer 1-4で40%を達成し、手動での重要案件補完で対応 |
| Claude API コスト超過 | 予算オーバー | Haiku利用でコスト最小化、キーワードフィルタで不要API呼び出し削減 |
